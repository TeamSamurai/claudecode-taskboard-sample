# CLAUDE.md

このファイルはClaude Codeがこのプロジェクトで作業する際のガイドラインを定義します。

## プロジェクト概要

タスクボードアプリケーション。

## Git 運用ルール

**コードを変更するたびに、必ずGitHubにプッシュすること。**

### 手順

1. 変更をステージング
   ```bash
   git add <変更したファイル>
   ```

2. コミット（意味のあるメッセージを付ける）
   ```bash
   git commit -m "変更内容の説明"
   ```

3. GitHubにプッシュ
   ```bash
   git push origin main
   ```

### コミットメッセージの規則

- 日本語または英語で記述する
- 変更の「何を」「なぜ」を簡潔に表現する
- 例: `feat: タスク追加機能を実装`, `fix: 削除ボタンが動作しないバグを修正`

### 注意事項

- `.env` などの機密ファイルはコミットしない
- 大きな変更は細かく分けてコミットする
- 破壊的な操作（force push等）はユーザーに確認を取ってから実行する

## コーディング規則

- 既存のコードスタイルに合わせる
- 不要なコメントや型アノテーションを追加しない
- 依頼された変更のみを行い、余分なリファクタリングはしない

## その他

- テストがある場合は変更後に必ず実行する
- セキュリティの脆弱性（SQLインジェクション、XSS等）を導入しない
